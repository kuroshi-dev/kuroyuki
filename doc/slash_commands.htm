Slash commands are invoked by typing / in the Discord message field. The system automatically shows available commands.

/help

/help [command]

Description: Shows a list of all available commands or detailed information about a specific command.

Parameters:

  • command (optional) - command name to get detailed information

Examples:

  • /help - shows all commands
  • /help userinfo - shows information about the userinfo command

/ping

/ping

Description: Checks the bot's connection latency with Discord.

Parameters: None

Result: Shows latency in milliseconds

Note: This command is useful for diagnosing connection problems.

/hello

/hello

Description: Bot greets the user with a personalized message.

/userinfo

/userinfo [user]

Description: Displays detailed information about a user.

Parameters:

  • user (optional) - user to check. If not specified, shows information about you.

Information displayed:

  • User ID
  • Account creation date
  • Server join date
  • Server nickname
  • Status (online/offline/do not disturb)
  • List of roles

/serverinfo

/serverinfo

Description: Shows detailed information about the current Discord server.

Parameters: None

Information displayed:

Field Description
ID Unique server identifier
Creation Date When the server was created
Member Count Total number of users
Channel Count All text and voice channels
Role Count All roles on the server
Emoji Count Custom server emojis
Owner Who created/owns the server
Boost Level Server Boost level (0-3)
Boost Count How many users boosted the server

/timeout

/timeout <member>

Description: Puts a user in timeout (temporarily blocks the ability to write).

Required permissions: Moderate Members

Parameters:

  • member (required) - user to timeout

Timeout duration:

  • 1 minute
  • 5 minutes
  • 10 minutes
  • 30 minutes
  • 1 hour
  • 1 day
  • 1 week
Important: You cannot timeout a user with higher permissions.

General Recommendations

  • All slash commands appear automatically when typing /
  • Required parameters are marked as <parameter>
  • Optional parameters are marked as [parameter]
  • Some commands may be unavailable depending on your server permissions
