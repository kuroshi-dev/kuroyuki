Command Prefix

All text commands start with the prefix - (hyphen). This prefix can be customized per server by server administrators.

Example: -ping, -help, -userinfo

Text commands provide a quick and familiar way to interact with Kuroyuki Bot. These commands are especially useful for experienced Discord users and can be faster to type than slash commands.

Information Commands

-help [command]

Description: Displays a list of all available commands or detailed information about a specific command.

Parameters:

  • command (optional) - name of a specific command to get details about

Examples:

  • -help - shows all available commands
  • -help ping - shows detailed information about the ping command

Aliases: None

-ping

Description: Checks the bot's latency and connection status with Discord.

Parameters: None

Output: Bot latency in milliseconds and API response time

Note: This command is useful for diagnosing connection issues and monitoring bot performance.

-hello

Description: Bot sends a friendly greeting message to the user.

Parameters: None

Example output: "Hello, Username! üëã"

Usage: Great for testing bot responsiveness and user interaction

-userinfo [user]

Description: Displays comprehensive information about a user's account and server membership.

Parameters:

  • user (optional) - user to check (mention, ID, or username). If not specified, shows your own information

Information displayed:

  • User ID and discriminator
  • Account creation date
  • Server join date
  • Server nickname (if set)
  • Current status and activity
  • List of assigned roles
  • Avatar and server avatar

Examples:

  • -userinfo - shows your information
  • -userinfo @Username - shows mentioned user's information
  • -userinfo 123456789012345678 - shows user by ID

-serverinfo

Description: Displays detailed information about the current Discord server.

Parameters: None

Information displayed:

Field Description
Server Name & ID Server name and unique identifier
Creation Date When the server was created
Owner Information Server owner details
Member Statistics Total members, online count, bot count
Channel Count Text, voice, and category channels
Role Count Number of roles in the server
Emoji Count Custom server emojis and stickers
Boost Information Boost level, boost count, and premium features
Verification Level Server security settings

Utility Commands

-commandlist

Description: Displays a comprehensive list of all available text commands with brief descriptions.

Parameters: None

Output: Categorized list of commands with usage syntax

Note: This command is equivalent to -help but provides a more compact view.

Administrative Commands

Restricted Access

The following commands are restricted to the bot owner only. These commands require elevated permissions and can affect bot operation.

-restart

Description: Restarts the bot process, reloading all modules and configurations.

Parameters: None

Access: Bot owner only

Warning: This command will temporarily disconnect the bot from Discord during the restart process.

Use cases:

  • Apply configuration changes
  • Recover from memory leaks
  • Update bot code in development

-shutdown

Description: Safely shuts down the bot, disconnecting from Discord and terminating the process.

Parameters: None

Access: Bot owner only

Warning: This command will completely stop the bot. Manual restart will be required.

Use cases:

  • Maintenance periods
  • Emergency shutdown
  • Server migration

-redeploy

Description: Redeploys all slash commands to Discord, updating command definitions and permissions.

Parameters: None

Access: Bot owner only

Note: This command may take a few minutes to complete. Slash commands might be temporarily unavailable during deployment.

Use cases:

  • After adding new slash commands
  • Updating command descriptions or parameters
  • Fixing command registration issues

Command Usage Guidelines

Best Practices

  • Case Sensitivity: Commands are case-insensitive (-PING works the same as -ping)
  • Parameter Formats: User mentions, IDs, and usernames are all accepted for user parameters
  • Error Handling: The bot provides helpful error messages for incorrect usage
  • Rate Limiting: Commands have built-in cooldowns to prevent spam

Common Parameter Types

Type Format Example Description
User Mention @username @JohnDoe Direct user mention
User ID 123456789012345678 123456789012345678 18-digit user ID
Username username johndoe Username without @

Error Messages

Common error messages you might encounter:

  • "Command not found": Check command spelling and prefix
  • "Insufficient permissions": You don't have required permissions
  • "User not found": The specified user doesn't exist or isn't in the server
  • "Missing parameters": Required parameters weren't provided
  • "Command on cooldown": Wait before using the command again

Comparison with Slash Commands

While text commands offer familiarity and speed, consider these differences:

  • Advantages: Faster for experienced users, work with automation, customizable prefix
  • Disadvantages: No auto-completion, require memorization, less discoverable for new users
  • Recommendation: Use slash commands for occasional use, text commands for frequent operations
