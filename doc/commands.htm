Kuroyuki Bot supports two types of commands to provide maximum flexibility and compatibility with different user preferences and Discord client versions.

Command Types Overview

‚ö° Slash Commands

Modern Discord command system introduced in 2021. These commands are triggered by typing / and provide auto-completion, built-in help, and better user experience.

  • Auto-completion support
  • Parameter validation
  • Built-in help text
  • Better security
  • Cross-platform compatibility

Usage: Type / in Discord chat

üí¨ Text Commands

Traditional prefix-based commands that have been used in Discord bots for years. These commands start with a prefix character (- by default).

  • Familiar to long-time Discord users
  • Quick to type for experienced users
  • Can be used in automated scripts
  • Works with older Discord clients
  • Customizable prefix per server

Usage: Type -command in Discord chat

üîß Administrative Commands (Text Only)

  • Restart: Restart the bot (owner only)
  • Shutdown: Safely shut down the bot (owner only)
  • Redeploy: Redeploy slash commands (owner only)
  • Command List: Display all available text commands

Command Availability

Most commands are available in both slash and text formats. However, some administrative commands are only available as text commands for security reasons.

Permission System

Commands have different permission requirements:

  • Public Commands: Available to all server members (help, ping, hello, userinfo, serverinfo)
  • Moderation Commands: Require "Moderate Members" permission (timeout)
  • Administrative Commands: Restricted to bot owner only (restart, shutdown, redeploy)

Important Notes

  • Some commands may not be available if the bot lacks necessary permissions
  • Administrative commands are restricted to the bot owner for security
  • Moderation commands respect Discord's role hierarchy
  • The bot cannot perform actions on users with higher roles

Command Syntax Convention

This documentation uses the following syntax conventions:

  • <parameter> - Required parameter
  • [parameter] - Optional parameter
  • parameter|option - Choice between options

Getting Help

For detailed information about any command:

  • Slash Commands: Use /help [command]
  • Text Commands: Use -help [command] or -commandlist
